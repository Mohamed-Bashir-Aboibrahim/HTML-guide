Document Structure

<!DOCTYPE>
Defines the document type and HTML version.
The <!DOCTYPE> declaration is not an HTML tag; it's an instruction to the web browser about what version of HTML the page is written in. It must be the very first thing in your HTML document.
<!DOCTYPE html>
<html>
Root element of an HTML document.
The <html> element represents the root of an HTML document. It is the container for all other HTML elements (except for the <!DOCTYPE> declaration). The lang attribute specifies the language of the document.
<html lang="en">
<!-- Document content -->
</html>
<head>
Container for metadata, scripts, and styles.
The <head> element contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets. It is placed between the <html> tag and the <body> tag.
<head>
  <title>Page Title</title>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="styles.css">
</head>
<body>
Container for the visible page content.
The <body> element contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc. There can only be one <body> element in an HTML document.
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>
</body>

Text Content

<h1> - <h6>
Heading elements from level 1 (most important) to level 6.
HTML headings are defined with the <h1> to <h6> tags. <h1> defines the most important heading. <h6> defines the least important heading. Search engines use headings to index the structure and content of your web pages.
<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Section Heading</h3>

<p>
Defines a paragraph of text.
The <p> tag defines a paragraph. Browsers automatically add a single blank line before and after each <p> element. Paragraphs are block-level elements, which means they take up the full width of their container.
<p>This is a paragraph of text. It contains sentences that form a coherent block of content.</p>
<p>This is another paragraph. Paragraphs are separated by whitespace.</p>

This is a paragraph of text. It contains sentences that form a coherent block of content.

This is another paragraph. Paragraphs are separated by whitespace.

<ul>, <ol>, <li>
Unordered and ordered lists with list items.
HTML lists allow web developers to group a set of related items:
  • <ul> - Defines an unordered list (bulleted)
  • <ol> - Defines an ordered list (numbered)
  • <li> - Defines a list item
<ul>
  <li>Unordered item 1</li>
  <li>Unordered item 2</li>
</ul>

<ol>
  <li>First ordered item</li>
  <li>Second ordered item</li>
</ol>
  • Unordered item 1
  • Unordered item 2
  1. First ordered item
  2. Second ordered item
<blockquote>
Defines a section quoted from another source.
The <blockquote> tag specifies a section that is quoted from another source. Browsers usually indent <blockquote> elements. The cite attribute can be used to specify the source URL of the quote.
<blockquote cite="http://example.com">
  This is a long quotation from another source.
</blockquote>

Links

<a>
Defines a hyperlink.
The <a> tag defines a hyperlink, which is used to link from one page to another. The most important attribute is href, which indicates the link's destination. By default, links appear as blue underlined text.

Key Attributes:

  • href - Specifies the URL of the page the link goes to
  • target - Specifies where to open the linked document
  • download - Specifies that the target will be downloaded
  • rel - Specifies the relationship to the linked document
<a href="https://example.com" target="_blank">Visit Example</a>
Visit Example
<nav>
Defines a set of navigation links.
The <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.
<nav>
  <a href="#home">Home</a>
  <a href="#about">About</a>
  <a href="#contact">Contact</a>
</nav>
Home About Contact

Multimedia

<img>
Embeds an image in the document.
The <img> tag is used to embed an image in an HTML page. It has two required attributes:
  • src - Specifies the path to the image
  • alt - Specifies an alternate text for the image
Images are not technically inserted into a web page; they are linked to web pages.
<img src="logo.png" alt="Company Logo" width="200" height="100">
<video>
Embeds video content.
The <video> tag is used to embed video content in a document. It may contain one or more video sources, represented using the <source> tag. The controls attribute adds video controls like play, pause, and volume.

Key Attributes:

  • controls - Displays video controls
  • autoplay - Video plays automatically
  • loop - Loops the video
  • muted - Mutes the audio
  • poster - Image to show while video is loading
<video controls width="400">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>
<audio>
Embeds sound content.
The <audio> tag is used to embed sound content in a document. It may contain one or more audio sources, represented using the <source> tag. The controls attribute adds audio controls like play, pause, and volume.

Key Attributes:

  • controls - Displays audio controls
  • autoplay - Audio plays automatically
  • loop - Loops the audio
  • muted - Mutes the audio
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Forms

<form>
Container for form elements.
The <form> element is used to create an HTML form for user input. It can contain various input elements like text fields, checkboxes, radio buttons, submit buttons, etc. Forms are used to collect user input which is sent to a server for processing.

Key Attributes:

  • action - Specifies where to send form data
  • method - HTTP method (get or post)
  • target - Where to display response (_blank, _self)
  • autocomplete - Enable/disable autocomplete
<form action="/submit" method="post">
  <!-- Form controls -->
</form>
<input>
Creates interactive controls.
The <input> element is the most important form element. It can be displayed in several ways, depending on the type attribute. Some common types include text, password, radio, checkbox, submit, button, email, and date.

Common Types:

  • text - Single-line text input
  • password - Password field (masked)
  • radio - Radio button (select one)
  • checkbox - Checkbox (select multiple)
  • submit - Submit button
  • button - Clickable button
  • email - Email address field
  • date - Date picker
<input type="text" name="username" placeholder="Enter username">
<input type="password" name="password" placeholder="Enter password">

<select>
Creates a drop-down list.
The <select> element is used to create a drop-down list. The <option> tags inside define the available options in the list. The <optgroup> element can be used to group related options.

Key Attributes:

  • name - Specifies name for the control
  • multiple - Allows multiple selections
  • required - Requires selection before submit
  • size - Number of visible options
<select name="country">
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
  <option value="ca">Canada</option>
</select>
<textarea>
Defines a multi-line text input control.
The <textarea> tag defines a multi-line text input control. It is often used in a form to collect user inputs like comments or reviews. A text area can hold an unlimited number of characters.

Key Attributes:

  • rows - Specifies the visible number of lines
  • cols - Specifies the visible width
  • placeholder - Specifies a hint
  • maxlength - Specifies max characters allowed
<textarea rows="4" cols="50" placeholder="Enter your message">
  Default text here
</textarea>

Tables

<table>
Defines an HTML table.
The <table> tag defines an HTML table. An HTML table consists of one <table> element and one or more <tr>, <th>, and <td> elements. Tables are useful for displaying tabular data.
<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Data 1</td>
    <td>Data 2</td>
  </tr>
</table>
Header 1 Header 2
Data 1 Data 2
<tr>, <td>, <th>
Table row, cell, and header cell elements.
These elements define the structure of an HTML table:
  • <tr> - Defines a table row
  • <td> - Defines a standard cell
  • <th> - Defines a header cell
The <th> element should be used for header cells that label rows or columns.
<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>John</td>
    <td>30</td>
  </tr>
</table>

Semantic HTML

<header>
Represents introductory content.
The <header> element represents a container for introductory content or a set of navigational links. A <header> typically contains:
  • One or more heading elements
  • Logo or icon
  • Authorship information
<header>
  <h1>Website Title</h1>
  <nav>
    <a href="#home">Home</a>
    <a href="#about">About</a>
  </nav>
</header>
<section>
Defines a section in a document.
The <section> element represents a standalone section of content that typically has a heading. Examples of sections could be chapters, tabbed content, or numbered sections of a paper.
Note: A section should make sense on its own and should be a logical part of the document outline.
<section>
  <h2>Section Title</h2>
  <p>Section content...</p>
</section>
<article>
Specifies independent, self-contained content.
The <article> element specifies independent, self-contained content. It should make sense on its own and it should be possible to distribute it independently from the rest of the site. Examples include:
  • Forum posts
  • Blog posts
  • News stories
  • Comments
<article>
  <h2>Article Title</h2>
  <p>Article content...</p>
</article>
<aside>
Defines content aside from the main content.
The <aside> element defines some content aside from the content it is placed in. The aside content should be indirectly related to the surrounding content. It is often placed as a sidebar in a document.
<aside>
  <h4>Related Content</h4>
  <p>Additional information...</p>
</aside>
